Web Servers

Client

The starting point for this website is VS Code's HTML boilerplate. The client used to view and test the website is Firefox on Linux.

Local Access

Accessing the website from a local computer must include the index.html resource in order to view the homepage. Not specifying the resource "/index.html" brings up the directory of the file. In Firefox's Inspector under Network, there is one GET request to an HTML document; the status code is 200.

Server Access

Server access need not include the index.html resource because the file is already the designated homepage of the web server. This file, stored in the root directory, is fetched with a GET request.

The Inspector shows a status code 200. This status code means that the request to the server to render the page in the client was successful. The data received is the HTML (text) of the file.

There are four GET requests; only two paths (homepage and favicon) are successful. The other two GET requests, to the JavaScript and CSS main files, are unsuccessful (status code 404) because these files do not exist. The favicon.ico request is successful. The response header shows the request has been cached by the DNS. Due to the cache, the status code is 200, even though no such file exists in the public_html folder on the server.

URL

The homepage URL is "https://web.engr.oregonstate.edu/~dushanol/a2-dushanol/index.html". The protocol is "HTTPS" or encrypted HTTP. There are two subdomains, "web" and "engr". The domain is "oregonstate.edu". The path to the resource is "~dushanol/a1-dushanol". And finally, the resource is "index.html" or the homepage.

Frontend Design

What is Frontend Design

Frontend design focuses on the interaction with the user. Often referred to as User Interface (UI) or User Experience (UX), this field focuses on visual design which enables usability.

Usability

A guide to reaching ultimate usability of a website consists of the following (known as the five Es):

Effective
Helps users meet their goals.
Efficient
Tasks are performed in the fewest number of steps.
Easy to navigate
First-time users, especially, locate their goals quickly.
Error-free
Study and remove roadblocks for specific users.
Engaging
Meet the needs of the intended audience.

Page Layout Tags

Page layout tags are used to section off content. Each tag usually has a content block so that a line break comes before and after.

  • <main> is the primary block that holds content.
  • <body> displays the contents of the browser's viewport and can appear only once on a page.
  • <section> is a thematic grouping of content with an <h1> heading.
  • <article> is a specific topic inside a <section> with <h2> heading.
  • <h1> through <h6> are hierarchical headlines, which allow screen readers and search engines access and use the page's structure.
  • <footer> is below the </main> tag and contains legal and contact information.

Anchors

Anchors create hyperlinks from one item to another can be created using <a> the anchor element. This element can hyperlink to:

  1. Internal content using IDs <article id="Web Servers">Web Servers/article>>, which results in Web Servers
  2. External content using <a href="https://github.com/">GitHub</a>, which results in GitHub
  3. Page-to-page sing <nav> and <a> before the <header>, which with a css file creates buttons to other pages.

Optimizing Images

Image optimization is a common practice in web development. Optimization reduces load time for images in the browser and conserves server space. There are six main optimization specifications: descriptive file naming, small file size, reducing dimensions, ensuring a correct file format, reducing the image resolution, and choosing an appropriate color mode.

Descriptive File Naming

Ideally, the name of the image file will include who, what, where, and when. Such titles improve search engine optimization, so search engine bots can categorize properly a given image on the web.

Small File Size

The smaller the image file size, the faster it will load. Known as compression, removing data image via approximations, in turn reduces the file size. Such compression, however, may result in pixelation and is known as lossy compression. Lossless compression, on the other hand, does not degrade the image quality, but it also does not compress the file size as much as lossy compression.

Reducing Dimensions

Images should be cropped to fit the spaces in a given web page. Most webpages comfortably fit a 600-pixel wide image.

Correct File Format

  • Online images are usually in .JPG (also .JPEG) format. JPEG stands for Joint Photographic Experts Group. It is a commonly used lossy form of compression, which came about in 1992.
  • Logos and icons are typically in .GIF format, which stands for Graphics Interchange Format, a bitmap image developed in the '80s.
  • Graphics, or designs on some surface, are typically in 24-bit .PNG format, which stands for Portable Network Graphics, a format which supports lossless compression.

Reducing the Resolution

How a picture shows up on a monitor is decided by the monitor's pixel-per-inch (ppi) support. Modern monitors support 300+ ppi, but because of variation, it is best to provide multiple image sizes.

Color Modes

The main color modes are: JPG, WebP, PNG, SVG, and GIF.

Favicons

Favicons, also known as touch icons, are a quick way to identify a browser tab. Favicons can be saved as .GIF, .PNG, .ICO, and .SVG files. To create a favicon, all that is needed is a symbol, which is modified to fit different browsers. A favicon is a square, so it is best to come up with a symbol that firs well within a square and is readable. The favicon for this website, for example, is a terminal with a ">" prompt and a made up command "l".

Favicons can be used to appear not only on browsers tabs, but also on a smartwatch, a cellphone, or even in engine search results. Because of all of these formats, a generator tool can produce appropriate formats for say Chrome on an Android device or Chrome on a personal computer.

Web Servers

Client

Starting point for this websute is VS Code's HTML boilerplate. The client used to view and test the website is Firefox on Linux.

Local Access

Accessing the website from a local computer must include the index.html resource in order to view the homepage. Not specifying the resource "/index.html" would result in viwing the directory of the file. In Firefox's Inspector under Network, there is one GET request to an HTML document; the status code is 200.

Server Access

Server access does not need to include index.html as a resource. The reason is that the file is already the designated homepage of the web server. This file, stored in the root directory, is fetched with a GET request.

Firefox's Inspector shows a status code 200. This status code means that the request to the server to render the page in the client was successful. The data received is the HTML (text) of my file.

There are four GET requests; only two (for the homepage and the favicon paths) are successful. The other two GET requests, to the JavaScript and CSS main files, are unsuccessful because these files do not exist. Thus, the 404 status code. The favicon.ico request is successful, even though I did not create such a file. The response header shows the request has been cached by the DNS. This is likely why the status code is 200, even though no such file exists in the public_html folder on the server, either.

URL

The URL for this website is "https://web.engr.oregonstate.edu/~dushanol/a1-dushanol/index.html". The protocol is "HTTPS" or encrypted HTTP. There are two subdomains, "web" and "engr". The domain is "oregonstate.edu". The path to the resource is "~dushanol/a1-dushanol". And finally, the resource is "index.html" or the homepage.

Frontend Design

What is Frontend Design

Frontend design focuses on the interaction with the user. Often referred to as User Interface (UI) or User Experience (UX), this field focuses on visual design which enables usability.

Usability

A guide to reaching ultimate usability of a website consists of the following (known as the the five Es):

Effective
Helps users meet their goals.
Efficient
Tasks are performed in the fewest number of steps.
Easy to navigate
First-time users, especially, locate their goals quickly.
Error-free
Study and remove roadblocks for specific users.
Engaging
Meet the needs of the intended audience.

Page Layout Tags

Page layout tags are used to section off content. Each tag usually has a content block so that a line break comes before and after.

  • <main> is the primary block that holds content.
  • <body> displays the contents of the browser's viewport and can appear only once on a page.
  • <section> is a thematic grouping of content with an <h1> heading.
  • <article> is a specific topic inside a <section> with <h2> heading.
  • <h1> through <h6> are hierarchical headlines which allow screen readers and search engines access and use the page's structure.
  • <footer> is below the </main> tag and contains legal and contact information.

Anchors

Hyperlinks from one item to another can be created using <a> the anchor element. This element can hyperlinks to:

  1. Internal content using IDs <article id="Web Servers">Web Servers/article>>, which results in
    Web Servers
  2. External content using <a href="https://github.com/">GitHub</a>, which results in GitHub
  3. Page-to-page sing <nav> and <a> before the <header>, which with a css file creates buttons to other pages.
